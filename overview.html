WORKFLOW

フェーズごとに「人の関わり方」を変える

一律に「毎回確認」でも「全部自動」でもありません。前提が決まる上流は人が厚く関わり、 前提が固まった中流は全自動、外部に影響する境界だけ人が意思決定します。

🔒 = 人のゲート承認が必要　🔍 = 独立レビュー（別コンテキストのAI）が必須通過点

FEATURES

「祈り」ではなく、仕組みで品質を守る

すべての特徴は一次情報の調査と実機検証、そして実プロジェクトのE2E検証に基づいています（根拠は DECISIONS.md に31件を記録）。

機械的ガードレール

テンプレート破壊・危険なgit操作・ハーネス自身の自己改変を、LLMの善意ではなくシェルフックで機械的にdeny/ask。Claude Codeでは permissions.deny との二重防御。

多層セキュリティチェック

リリース前に8ステップのセキュリティ監査（OWASP Top10・依存脆弱性・データフロー追跡。GitHub公式skill由来）を必ず通過。シークレットのハードコードはフックが書き込んだ瞬間に検知してdeny。

独立レビュー（別セッション）

書いた本人に自己承認させない。要件・設計は spec-critic、コードは reviewer が、まっさらな独立コンテキストでゲート前に監査。上流ほど手戻りが高くつくから、レビューは上流から。

必要なスキルを自ら作る

未知のデプロイ環境・技術スタックに遭遇したら、その場で手順をSkillとして自作（deploy-gas等）。公式・コミュニティ製の流用を優先し、2回目からは自動化された状態で始まる。ハーネスに全環境を事前収録しない設計。

UIデザインゲート

画面は実装前に自己完結型HTMLモックアップで目で見て承認。閲覧はブラウザで開くだけ＝トークンゼロ。「できて動かして初めてデザインが分かる」を構造的に防ぐ。

コンテキストロット対策

長い会話ほどAIの精度は落ちる。docs/ が記憶、会話は使い捨て。実装は task-worker が1タスク=1独立コンテキストで実行し、セッション分割基準も数値で定義済み。

成長ループ

教訓は learnings.md に1行追記→全セッションに自動注入。リリース後の振り返りが改善提案表を生み、本体テンプレートへ還流。次のプロジェクトは今日の学びから始まる。

3プラットフォーム対応

振る舞いの正は1か所（AGENTS.md + .github/）。各環境にはポインタだけの薄いアダプタを機械生成。Copilot / Claude Code / Antigravity のどこで開いても /00-start-project から同じ体験。

要件定義の専門性

Airbus・NASAが使うEARS記法で曖昧語を排除、NFRは「測れる文」に定量化、前提・リスク台帳、デプロイ環境と自動化境界の聞き切り。後工程の全自動はこの厚さが支える。

小規模から大規模・既存まで

単一パイプライン（小〜中）／サブシステム分割＋インターフェース契約ICD（大規模）／as-is逆起こし（既存コードベース）。規模はエージェントが判定して構造を提案。

トークンコスト最適化

高頻度エージェントは model:auto、強モデルは設計とレビューだけ。ブラウザ検証はコード化テスト主（対話型MCPの約1/4のコスト）。スキルは段階的開示で必要な時だけ読み込み。

差分駆動の保守

リリース後の改修で全フェーズをやり直さない。変更部分だけ再ゲートし、影響はトレーサビリティ表から辿る。緊急ホットフィックスは「記録すれば許容」——現実に破られないルール設計。

ARCHITECTURE

正は1か所、アダプタは薄く

振る舞いの定義はすべて正のレイヤに置き、各プラットフォームには「正を読んで実行せよ」という ポインタだけを機械生成します。だからドリフトせず、改善は1回で3環境に届きます。

HUMAN × AI

どこまでAIに任せ、どこを人が守るか

原則はひとつ——内側はAI、境界は人。誤りが全体に波及する意思決定にだけ、人の注意を集中させます。

フェーズ / 作業担当人がやること
要件定義（ヒアリング・文書化）協働質問に厚く答える＋ゲート承認
アーキテクチャ・詳細設計AI技術選定の相談への回答＋最終承認1回
UIモックアップの確認人ブラウザで開いて見る（トークンゼロ）
実装〜テスト〜自動修復AI基本ノータッチ（真のブロッカーのみ）
独立レビュー＋セキュリティ監査AI—（別コンテキストのAIが自動監査）
サブシステム分割・ICD変更（大規模）人必ず承認（最重要ゲート）
git push / タグ / デプロイ人フックが確認を強制。最終意思決定
振り返り→ハーネス改善の適用協働AIが提案表を生成、人が本体へ還流
